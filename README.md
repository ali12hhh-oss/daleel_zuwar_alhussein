# دليل زوار الحسين — مشروع Flutter

تطبيق موبايل ديني حسيني يتضمن الأقسام التالية:
1. **طريق زائر الحسين**: تحديد موقع المستخدم عبر GPS وحساب المسافة إلى كربلاء المقدسة، مع فتح مسار مشي فعلي عبر خرائط جوجل، بالإضافة لقائمة بأبرز مدن العراق ومسافاتها التقريبية.
2. **أسئلة شرعية**: اختيار المرجع الديني الشيعي (السيستاني، الشهيد الصدر، الخامنئي، الشيرازي، الخوئي) مع زر ينقل المستخدم مباشرة إلى صفحة البحث في المسائل الشرعية (الاستفتاءات) في الموقع الرسمي لكل مرجع.
3. **أقوال الإمام الحسين عليه السلام**: خطبه وكلماته في كربلاء.
4. **خطب السبايا**: خطب أهل بيت الإمام الحسين السبايا من كربلاء إلى الكوفة والشام وحتى رجوعهم إلى المدينة المنورة.
5. **مودة أهل البيت عليهم السلام**: أحاديث نبوية مشهورة.
6. **معركة الطف**: جدول زمني لأحداث الأيام العشرة من محرم.
7. **ولادات ووفيات أهل البيت**: تبويب للولادات وتبويب للوفيات/الاستشهادات لأهل البيت الأربعة عشر معصوماً (إضافة إلى السيدة زينب وأبي الفضل العباس عليهما السلام) بالتاريخ الهجري وسبب الوفاة، حسب مصادر شيعية.
8. **وضع نهاري / ليلي**: زر تبديل في الشاشة الرئيسية (أعلى يمين) للتنقل بين المظهرين.
9. **أيقونة تطبيق مخصصة**: تصميم قبة ذهبية على خلفية خضراء (ملف `assets/icon/app_icon.png`) مُعدّة تلقائياً لتصبح أيقونة التطبيق عبر حزمة flutter_launcher_icons.

## ⚠️ تنبيهات مهمة قبل النشر (اقرأها أولاً)

هذا المشروع هو **هيكل تطبيق كامل وجاهز للعمل**، لكنه يحتاج إلى تدقيق محتوى ديني قبل نشره:

- **قسم الأسئلة الشرعية**: لم أقم بإدراج أي فتوى فعلية، لأن الفتاوى الدقيقة تتغيّر وتختلف حسب صياغة السؤال، وتنسب مباشرة لأشخاص حقيقيين، وأي خطأ في نقلها قد يُعد تضليلاً دينياً خطيراً. الملف `lib/data/questions_data.dart` يحتوي على نماذج فارغة (Placeholder) فقط، يجب تعبئتها بأجوبة حرفية منقولة من المواقع الرسمية لكل مرجع مع ذكر رابط المصدر وتاريخه:
  - السيد السيستاني: https://www.sistani.org
  - السيد الخامنئي: https://arabic.khamenei.ir
  - يفضَّل التواصل مع مكاتب بقية المراجع (الشهيد الصدر، الشيرازي، الخوئي) للحصول على المصدر الرسمي المعتمد لديهم.
- **أقوال الإمام الحسين ومودة أهل البيت** (`lib/data/hussein_quotes_data.dart`): أدرجت نماذج من أشهر الأقوال والأحاديث المعروفة تاريخياً مع ذكر مصدرها العام، لكن يُنصح بمراجعتها حرفياً من الكتب الأصلية (اللهوف، مقتل المقرم، بحار الأنوار) قبل الاعتماد النهائي.
- **معركة الطف** (`lib/data/battle_data.dart`): ملخص عام بأسلوب معاصر لأحداث الأيام العشرة، يُنصح بمراجعته وتوسيعه من المصادر التاريخية المعتمدة.
- **خطب السبايا** (`lib/data/sabaya_data.dart`): مقتطفات من خطب مشهورة (زينب عليها السلام في الكوفة والشام، زين العابدين عليه السلام في الكوفة والشام، أم كلثوم عليها السلام)، مصدرها العام: بلاغات النساء، اللهوف، مقتل المقرم. يُنصح بمراجعة النص الكامل وتوسيعه بخطب إضافية (كخطبة فاطمة الصغرى، وخطب أخرى وردت في المصادر) قبل النشر النهائي.
- **ولادات ووفيات أهل البيت** (`lib/data/ahlulbayt_dates_data.dart`): تم الآن ذكر **جميع الروايات المعروفة** لكل تاريخ مختلف فيه (مثل استشهاد السيدة فاطمة الزهراء وميلاد ووفاة السيدة زينب وميلاد ووفاة الإمام زين العابدين وميلاد الإمام العسكري ووفاة الإمام الرضا عليهم السلام)، مع إسناد كل رواية إلى قائلها أو المصدر الذي نقلها، وتمييز الرواية الأشهر بنجمة ذهبية. الأحداث التي لم تجد فيها روايات متعددة موثقة تحمل رواية واحدة فقط منسوبة إلى "المشهور عند علماء الإمامية" — يُنصح بالتحقق من أي روايات إضافية عبر مصادر متخصصة (كموسوعات التراجم أو مفاتيح الجنان) إن رغبت بمزيد من الشمولية.
- **المسافات والمسارات** (`lib/data/cities_data.dart`): أرقام تقريبية للاسترشاد. المسار الفعلي على الخريطة يُفتح مباشرة عبر خرائط جوجل بدقة حقيقية.

## ✅ مجلد android/ مُضمَّن الآن كاملاً في المشروع

بعد ملاحظتك، أضفت مجلد `android/` كاملاً ومكتوباً يدوياً (وليس مولَّداً تلقائياً
عبر `flutter create`)، بمكتبات وإعدادات حديثة:

- **Gradle 8.7** عبر Kotlin DSL (`build.gradle.kts` بدل الصيغة القديمة Groovy).
- **Android Gradle Plugin 8.5.1** و **Kotlin 1.9.24**.
- **Java 17** كإعداد توافقية.
- **minSdk 23** و compileSdk/targetSdk يتبعان أحدث إصدار يدعمه Flutter تلقائياً.
- أذونات الموقع (GPS) والإنترنت مضافة مباشرة في `AndroidManifest.xml` منذ
  البداية — **لا حاجة لأي خطوة يدوية أو تلقائية لإضافتها لاحقاً**.
- اسم التطبيق بالعربية ("دليل زوار الحسين") ومعرّف الحزمة
  `com.daleelzuwar.alhussein` مضبوطان مسبقاً.
- أيقونة التطبيق (القبة الذهبية) مُولَّدة بكل الأحجام ومضمَّنة فعلياً في
  `android/app/src/main/res/mipmap-*/ic_launcher.png` — تعمل من أول بناء
  دون انتظار أي خطوة توليد إضافية.
- شاشة إقلاع (Splash) بلون أخضر العلامة التجارية.

**الاستثناء الوحيد** غير المُضمَّن هو ملفات bootstrap الخاصة بـ Gradle Wrapper
نفسها (`gradlew`, `gradlew.bat`, `gradle-wrapper.jar`) — وهذه ليست جزءاً من
كودك أو تخصيصاتك، بل هي سكربتات قياسية تُنزّل نسخة Gradle المناسبة تلقائياً؛
GitHub Actions يولّدها في كل بناء بأمر `gradle wrapper --gradle-version 8.7`
(خطوة "بناء سكربتات gradlew" في الـ workflow)، وهذا لا يمسّ أياً من ملفاتك
المخصصة إطلاقاً. إن رغبت بالتطوير محلياً، شغّل نفس الأمر مرة واحدة داخل
مجلد `android/` (يتطلب تثبيت Gradle على جهازك، أو استخدم Android Studio
الذي يقوم بذلك تلقائياً عند فتح المشروع).

## 🚀 بناء APK تلقائياً عبر GitHub (الطريقة الموصى بها — بدون تثبيت Flutter على جهازك)

أضفت للمشروع ملف Workflow جاهز في `.github/workflows/build-apk.yml` يقوم تلقائياً
بتثبيت Flutter وبناء ملف APK حقيقي على خوادم GitHub، دون أن تحتاج لتثبيت أي شيء
على جهازك. الخطوات:

1. أنشئ حساباً على https://github.com إن لم يكن لديك واحد.
2. أنشئ مستودعاً (Repository) جديداً فارغاً، مثلاً باسم `daleel-zuwar-alhussein`.
3. ارفع محتويات هذا المشروع إلى المستودع (عبر الموقع مباشرة بالسحب والإفلات
   drag & drop لكل الملفات، أو عبر الأوامر التالية إن كان لديك git مثبتاً):

```bash
cd daleel_zuwar_alhussein
git init
git add .
git commit -m "أول نسخة من تطبيق دليل زوار الحسين"
git branch -M main
git remote add origin https://github.com/USERNAME/daleel-zuwar-alhussein.git
git push -u origin main
```

4. بمجرد الرفع (Push)، سيبدأ GitHub تلقائياً بتشغيل عملية البناء. لمتابعتها:
   افتح المستودع في GitHub → تبويب **Actions** → ستجد عملية باسم "بناء تطبيق
   دليل زوار الحسين (APK)" قيد التنفيذ (تستغرق عادة 5-10 دقائق).
5. بعد اكتمال البناء بنجاح (علامة ✅ خضراء)، يمكنك الحصول على ملف الـ APK من
   مكانين:
   - تبويب **Actions** → افتح عملية البناء المكتملة → في الأسفل قسم
     **Artifacts** → حمّل `daleel-zuwar-alhussein-apk`.
   - أو تبويب **Releases** على يمين صفحة المستودع الرئيسية، حيث يُنشئ
     الـ Workflow إصداراً جديداً تلقائياً مع ملف الـ APK جاهزاً للتحميل
     المباشر في كل مرة ترفع فيها تعديلاً.
6. انقل ملف الـ APK إلى هاتفك الأندرويد وثبّته (فعّل خيار "التثبيت من مصادر
   غير معروفة" في إعدادات الهاتف إن طُلب منك ذلك).

بهذه الطريقة تحصل على تطبيق حقيقي قابل للتثبيت دون الحاجة لتثبيت Flutter SDK
على جهازك الشخصي. وإن رغبت لاحقاً بالتطوير محلياً، تبقى الطريقة التقليدية
أدناه متاحة أيضاً.

## المتطلبات لتشغيل المشروع محلياً (اختياري)

- تثبيت Flutter SDK (الإصدار 3.x) على جهازك: https://docs.flutter.dev/get-started/install
- محرر مثل Android Studio أو VS Code
- جهاز أندرويد أو محاكي لتجربة التطبيق

## خطوات التشغيل

```bash
# 1. فك ضغط المشروع والدخول إلى مجلده
cd daleel_zuwar_alhussein

# 2. تحميل الحزم (مجلد android/ موجود مسبقاً، لا حاجة لـ flutter create)
flutter pub get

# 3. تشغيل التطبيق على جهاز/محاكي متصل
flutter run
```

عند أول تشغيل، سيقوم Flutter أو Android Studio تلقائياً بتوليد ملفات
Gradle Wrapper (`gradlew`) المفقودة عمداً (راجع القسم أعلاه). إن واجهت
خطأ متعلقاً بـ gradlew، شغّل يدوياً داخل مجلد `android/`:
```bash
gradle wrapper --gradle-version 8.7 --distribution-type all
```

## توليد أيقونة التطبيق

الأيقونة **مُضمَّنة فعلياً** في `android/app/src/main/res/mipmap-*/` ولا
تحتاج أي خطوة إضافية. إن عدّلت تصميم الأيقونة لاحقاً (`assets/icon/app_icon.png`)
وأردت تحديثها، شغّل:

```bash
dart run flutter_launcher_icons
```

## بناء ملف APK حقيقي

بعد التأكد من عمل التطبيق:

```bash
flutter build apk --release
```

سيكون ملف الـ APK في المسار:
`build/app/outputs/flutter-apk/app-release.apk`

## التوقيع للنشر الفعلي على متجر جوجل بلاي (اختياري)

نسخة الإصدار (release) حالياً موقّعة بمفتاح التصحيح الافتراضي (debug) لتسهيل
أول بناء تجريبي وتثبيته يدوياً. لنشر التطبيق فعلياً على Google Play، يجب
إنشاء مفتاح توقيع خاص بك (Keystore) وربطه في `android/app/build.gradle.kts`
ضمن `signingConfigs`، بدلاً من `signingConfig = signingConfigs.getByName("debug")`.
راجع الدليل الرسمي: https://docs.flutter.dev/deployment/android#signing-the-app

## دعم اللغة العربية الكامل (اختياري لكن موصى به)

أضف إلى `pubspec.yaml` ضمن dependencies:
```yaml
flutter_localizations:
  sdk: flutter
```
ثم في `lib/main.dart` استورد `flutter_localizations` وأضف:
```dart
localizationsDelegates: const [
  GlobalMaterialLocalizations.delegate,
  GlobalWidgetsLocalizations.delegate,
  GlobalCupertinoLocalizations.delegate,
],
```

## تطوير قسم "طريق زائر الحسين" لاحقاً

- المسافة الحالية محسوبة كخط مستقيم (Haversine) للعرض السريع، وزر "عرض مسار المشي" يفتح خرائط جوجل بمسار مشي حقيقي ودقيق يتحدث تلقائياً.
- لعرض مسار مشي مفصّل *داخل* التطبيق نفسه (بدون الخروج لتطبيق خرائط)، يلزم ربط المشروع بمفتاح Google Directions API أو Maps SDK for Flutter (`google_maps_flutter`) — غير مُفعّل حالياً لتفادي الحاجة لمفاتيح API مدفوعة.
